Как мы измеряем доступность

Прозрачность о технологиях и методах замеров на этой странице.

Технологии

Страница работает полностью в браузере, без собственного сервера. Это важно, потому что ограничивает, какие проверки мы можем делать.

  • • Rust — язык, на котором написана логика.
  • • WebAssembly (WASM) — Rust скомпилирован в WASM и выполняется в браузере.
  • • Leptos — реактивный фреймворк для UI на Rust.
  • • Tailwind CSS — стилизация.

Доступность сервисов

Проверка доступности сторонних сервисов (YouTube, X/Twitter, Instagram, Telegram и др.) вынесена в браузерное расширение. Связано это с тем, что из обычного веб-страницы нельзя корректно отличить настоящий сайт от провайдерской заглушки: браузер не даёт прочитать тело ответа чужого домена без CORS.

Расширение работает из браузера пользователя, может читать HTTP-статус и тело ответа, поэтому определяет блокировки и заглушки точнее, чем статус-страница.

На этой странице оставлены проверки, которые не требуют чтения чужих ответов:

  • • Определение внешнего IP и VPN.
  • • Статус серверов FRKN и API api.frkn.org (CORS настроен).
  • • Измерение скорости загрузки файла.

Определение VPN

Мы определяем ваш внешний IP, страну и провайдера через публичные геолокационные сервисы: ipapi.co, ipwho.is и fallback на ipinfo.io.

VPN считается включённым, если:

  • • Провайдер (организация владельца IP) или ASN похожи на хостинг, дата-центр или VPN-сервис.
  • • Ваш внешний IP совпадает с одним из IP-адресов серверов FRKN, полученных в ответе API FRKN (https://api.frkn.org/nodes).

Серверы FRKN

Страница загружает актуальный список нод с https://api.frkn.org/nodes и параллельно проверяет доступность каждого hostname, а также самого API через тот же endpoint /nodes.

Сводный статус может быть:

  • • Все доступны — все ноды и API отвечают.
  • • Частично доступны — часть нод или API не отвечает.
  • • Недоступны — ни одна из проверенных точек не отвечает.

Если список нод не удалось загрузить (например, из-за отсутствия CORS на стороне API), карточка покажет статус Не удалось загрузить.

Измерение скорости

Speed test загружает тестовый бинарный файл фиксированного размера. Мы замеряем время от начала загрузки до получения всех байт и считаем скорость:

Mbps = (размер_файла_в_байтах × 8) / время_в_секундах / 1 000 000

К URL добавляется уникальный параметр ?cb=..., чтобы браузер не брал файл из кэша. Если 10 МБ файл недоступен или слишком медленный, пробуем 1 МБ.

Важные ограничения

  • • Это не полноценный uptime-мониторинг. Страница показывает состояние из вашего браузера и вашей сети прямо сейчас.
  • • Браузер не умеет ICMP ping. Пинг на этой странице — это HTTP-латентность.
  • • Проверка сторонних сервисов (YouTube, X/Twitter, Instagram и др.) вынесена в браузерное расширение, потому что без CORS нельзя отличить настоящий сайт от заглушки.
  • • Карточка FRKN загружает список нод с api.frkn.org/nodes. Если API не отдаёт CORS-заголовки, список не загрузится.
